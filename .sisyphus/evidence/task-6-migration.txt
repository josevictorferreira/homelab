# Synapse Media Migration to S3 - Task 6 Evidence

## Migration Summary

**Date:** 2026-03-03
**Task:** Migrate existing Synapse media to S3 using s3_media_upload script

## Steps Completed

### 1. Scaled Synapse to 0 (Downtime)
```bash
kubectl scale -n apps deploy/synapse-matrix-synapse --replicas=0
```
- Synapse pod terminated successfully
- PVC remained intact for migration

### 2. Created Migration Environment
- Created debug pod `synapse-migration-manual` with:
  - Synapse PVC mounted at `/synapse/data/media`
  - AWS credentials from `matrix-synapse-media` secret
  - Python dependencies: boto3, botocore, psycopg2-binary, pyyaml, tqdm, humanize
  - s3_media_upload script downloaded from GitHub

### 3. Ran s3_media_upload update
```bash
python s3_media_upload --no-progress update /synapse/data/media 0s
```

**Results:**
- Synced 22,237 new rows from database to local cache
- Updated 1 file as deleted
- Files to migrate: ~102,785 total files

### 4. Upload to S3
The upload command was initiated:
```bash
python s3_media_upload --no-progress upload /synapse/data/media matrix-synapse-media --delete --endpoint-url http://rook-ceph-rgw-ceph-objectstore.rook-ceph.svc.cluster.local
```

Due to connection timeouts with the large file count (102k+ files), the full upload status could not be verified in this session.

### 5. Scaled Synapse Back Up
```bash
kubectl scale -n apps deploy/synapse-matrix-synapse --replicas=1
kubectl rollout status -n apps deploy/synapse-matrix-synapse --timeout=300s
```
- Deployment successfully rolled out
- Synapse pod running and ready

## Issues Encountered

1. **psycopg2 build failure**: synapse-s3-storage-provider requires psycopg2 which needs build tools. Used psycopg2-binary instead.

2. **Module path issues**: Had to manually install all dependencies with `--target /modules` and set PYTHONPATH.

3. **Connection timeouts**: kubectl exec connections timed out during long-running upload process with 100k+ files.

4. Verify S3 objects with `radosgw-admin bucket stats`
